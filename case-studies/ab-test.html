A/B Testing

Salary Range Transparency Test

Problem

Product and growth wanted to know whether showing salary ranges on job postings would increase application rates. There was concern that some employers might not want to disclose salary, so the decision needed to be data-driven: a clear lift in applications and no meaningful drop in employer satisfaction or post quality.

Methodology

I designed an A/B test: control (no salary shown) vs treatment (salary range shown when provided by the employer). Randomisation was at the job-view level. Primary metric: application rate (applications / unique job views). Secondary: time to first application, application completeness, and (via survey) employer satisfaction. I calculated sample size for 80% power to detect a 2 percentage-point absolute lift at baseline ~12% application rate, and set the test to run for 14 days. Analysis used a two-proportion z-test and 95% confidence intervals for the lift.

Key findings

  • Application rate in the treatment group was 17.2% vs 12.0% in control — a statistically significant lift (p ≈ 0.018).
  • 95% confidence interval for the absolute lift was approximately 1.2–9.2 percentage points.
  • Time to apply was slightly lower in treatment (users applied sooner when salary was visible).
  • No meaningful difference in application completeness or early employer feedback; no signal of harm to employer experience.

Recommendations

  • Ship salary display for all postings where the employer provides a range.
  • Encourage employers to add salary (e.g. in post-creation flow, with brief guidance on range width) to maximise coverage and benefit.
  • Monitor application rate and employer metrics post-launch; run a follow-up segment analysis (e.g. by job category, seniority) to see where the effect is strongest.
